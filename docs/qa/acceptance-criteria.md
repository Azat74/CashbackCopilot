# Acceptance Criteria

## Setup wallet

- пользователь может добавить банк
- пользователь может добавить способ оплаты к банку
- пользователь может создать правило кешбека с категорией и reward model
- отрицательные значения для лимитов не принимаются

## Decide before paying

- пользователь может ввести сумму
- пользователь может выбрать категорию вручную
- пользователь может открыть scanner и безопасно вернуться к ручному сценарию
- если есть валидные варианты, показывается best option
- если валидных вариантов нет, показывается нейтральный empty state

## Trust and explainability

- recommendation содержит reasons
- uncertain recommendation содержит risks
- пользователь может изменить категорию и пересчитать результат

## Track outcomes

- пользователь может отметить факт оплаты
- история сохраняет recommended и actual method
- expected и actual cashback можно сравнить
