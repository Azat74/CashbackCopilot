# Business Logic Guardrails

## Назначение

Этот документ фиксирует продуктовые и доменные инварианты, которые нельзя нарушать при развитии `Cashback Copilot`.

## Базовая позиция продукта

Приложение выдает рекомендацию перед оплатой. Оно не гарантирует фактическое начисление кешбека банком.

## Guardrails

1. Рекомендации advisory, а не authoritative.
2. `QR != MCC truth`; QR и merchant hints — это сигналы, а не доказательство банковой классификации.
3. Неопределенность должна быть видима в UI через confidence и risks.
4. Канал оплаты обязан учитываться явно.
5. Лимиты и минимальная сумма проверяются до выдачи рекомендации.
6. При неоднозначности расчет должен быть консервативным.
7. Raw QR payload не становится долговременным источником доменной истины.
8. История приложения не равна банковской выписке.
9. Продукт не хранит банковские креды, PAN и CVV.
10. Нельзя silently отправлять детальные платежные данные наружу.

## Recommendation invariants

- `expectedReward >= 0`
- `confidence` должен быть в диапазоне `[0, 1]`
- правило не может применяться к неподдерживаемому каналу
- правило не может применяться, если сумма меньше `minAmount`
- если валидных вариантов нет, приложение должно вернуть пустой результат, а не synthetic winner

## UX invariants

- каждая рекомендация содержит reward, confidence и reasons
- если есть uncertainty, пользователь видит risks без дополнительных действий
- пользователь может скорректировать категорию перед окончательной рекомендацией
