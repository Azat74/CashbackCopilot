# Vision

`Cashback Copilot` помогает пользователю за несколько секунд до оплаты понять, каким способом платить выгоднее всего.

Приложение сопоставляет пользовательские правила кешбека, лимиты, ограничения по каналу оплаты и контекст покупки, а затем выдает понятную и объяснимую рекомендацию.

Продукт строится вокруг purchase-first UX:

purchase -> recommendation -> payment

Пользователь думает о покупке, а не о таблице карт и категорий. Поэтому главный UX-барьер для продукта не в самом recommendation engine, а в том, насколько быстро пользователь может:

1. загрузить актуальные категории месяца
2. запустить сценарий рекомендации
3. получить ответ за 1-2 секунды

Чтобы recommendation engine был реально полезен, настройка правил не должна требовать ручного ввода десятков полей. Поэтому важная часть видения продукта — быстрый ежемесячный импорт категорий кешбека по скриншотам экранов банка с последующей проверкой черновика пользователем.

Это не банковский агрегатор, не финансовый трекер и не система гарантированного расчета кешбека. Основная ценность продукта — быстрый и надежный decision support в момент оплаты.

Три ключевых столпа MVP:

1. screenshot cashback import
2. recommendation engine
3. quick purchase UI

Целевой цикл использования:

1. В начале месяца пользователь импортирует категории кешбека по скриншотам.
2. Приложение локально создает черновик месячного набора правил и помечает неуверенные поля.
3. Пользователь быстро валидирует черновик и сохраняет его на текущий месяц.
4. Перед оплатой приложение рекомендует лучший способ оплаты с учетом актуальных правил месяца.
5. При необходимости пользователь уточняет сумму, merchant или QR-контекст, не теряя быстрый путь к рекомендации.
